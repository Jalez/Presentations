So, first of all, what is the DOM? The DOM is an in-memory representation of web documents like HTML and XML. It enables dynamic manipulation of web pages and interaction with users, by playing a critical role in bridging the gap between HTML/XML documents and JavaScript.

It's important to understand that the DOM is not part of the JavaScript language itself, but rather a programming interface that is commonly used alongside JavaScript to access and modify the content of web pages.

You could just as easily use the DOM with other programming languages, such as Python or Java. However, since JavaScript is the most commonly used language for web development, it's also the most common language to use with the DOM.

One of the key aspects of the DOM is its tree-like hierarchical structure, which represents documents as a collection of nodes and objects. This organization enables developers to easily traverse and manipulate elements within a document, allowing for efficient modification of web page content.

The DOM provides a wide range of functionalities that help developers create more interactive and user-friendly web applications. One such functionality is event handling, which allows developers to create responses to specific user actions, such as clicks or keypresses. Additionally, the DOM enables modification of elements and attributes, empowering developers to create dynamic, data-driven web applications that cater to user needs and preferences.

In summary, the DOM is an essential concept for web developers, acting as the bridge between HTML (or XML) documents and JavaScript. Its hierarchical structure and the functionalities it offers allow developers to create more engaging, interactive, and user-friendly web applications.

Understanding the DOM Structure

The DOM is a tree-like representation of an HTML document, where each element in the document is represented by a node in the tree.

The most important nodes

  • Root node: The document itself 
    const root = document
  • Element nodes: HTML elements, such as <body>, <header> <p> <div> <img> or <div> 
    const body = document.body // returns the body element
                const p = document.querySelector('p') // returns the first p element
  • Attribute nodes: Attributes, such as class, src and id, of HTML elements 
    const pClass = p.getAttribute('class') // returns the value of the class attribute (e.g. 'paragraph')
                  p.setAttribute('class', 'new-class') // sets the value of the class attribute to 'new-class'
  • Text nodes: Text content within HTML elements 
    const pText = p.textContent // returns the text content of the p element
                    console.log(pText) // 'A simple paragraph.'
                    p.textContent = 'New text' // sets the text content of the p element to 'New text'

Let's delve deeper the tree structure. As we saw in the previous slide, the DOM is made up of a collection of nodes, which are the basic building blocks of the DOM tree. Each node represents a different type of object within the document, such as an HTML element, an attribute, or a comment.

To better understand the DOM, we must first familiarize ourselves with the various types of nodes that make up its structure. These nodes can be classified into four main categories:

  • The root node, also known as the document node, represents the entire document itself.

    This node serves as the starting point of the DOM tree and encompasses all other nodes within the document. In an HTML document, the root node is the document object, which represents the HTML document as a whole.

  • Then we have the Element nodes, which correspond to HTML elements such as body, header, or div. These nodes serve as the building blocks of a web page's structure and directly affect the visual layout and organization of the content.

  • Attribute nodes, which represent the attributes of HTML elements, such as class and id. These nodes provide additional information about the elements they are associated with and can be used for styling purposes or to uniquely identify specific elements on a page.

    Attribute nodes are properties of element nodes, and can be accessed and manipulated using JavaScript. For example, you can get the value of an attribute using the getAttribute() method, change the value of an attribute using the setAttribute() method, or remove an attribute using the removeAttribute() method. More about these methods as well as other DOM methods later.

  • We also have Text nodes, which contain the actual text content within HTML elements. Text nodes are always children of an element node, and cannot have any child nodes of their own. They can be accessed and manipulated using JavaScript to change their content.

The less important nodes

  • Comment nodes: Comments within HTML documents (<!-- comment -- >) 
    // Create a new comment node with the text "This is a comment"
                  const newCommentNode = document.createComment('This is a comment'); 
                  // Add the new comment node to the document
                  document.body.appendChild(newCommentNode);
  • Document fragment nodes: A subset of a document 
                    const fragment = document.createDocumentFragment()//Create a new document fragment
                  const newP = document.createElement('p') //Create a new paragraph element
                  newP.textContent = 'A new paragraph' //Set the text content of the paragraph element
                  fragment.appendChild(newP) //Append the paragraph element to the document fragment
                  document.body.appendChild(fragment) //Append the paragraph element to the document
  • Document type nodes: The document type declaration (<!DOCTYPE html>) 
    const doctype = document.doctype //Returns the document type declaration
                  console.log(doctype) // <!DOCTYPE html>

Less known/used node types

Now, there are additional node types that may be of interest, although they are less commonly used in everyday web development. These include:

  • Comment nodes, which represent comments within an HTML document. These nodes are not displayed in the rendered page, but can be accessed and manipulated using JavaScript.

  • There are also these Document fragment nodes, which represent a subset of a document. A document fragment node represents a collection of nodes that are not part of a complete document. It can be used to group a set of nodes together before adding them to the DOM tree. Document fragment nodes do not have any parent nodes and are not part of the DOM tree until they are added to it using the appendChild() or insertBefore() method. Even then, only the child nodes of the document fragment node are added to the DOM tree. The created document fragment node itself becomes empty after it is appended to the body is because the appendChild() method used to append the fragment to the body actually moves the child nodes from the fragment to the parent node.

  • Document type nodes, which represent the document type declaration (DTD/DOCTYPE) of an HTML document. These nodes are not displayed in the rendered page, but can be accessed and manipulated using JavaScript.

DOM class hierarchy

The DOM class hierarchy is a tree structure that defines the relationships between the different interfaces in the DOM API.

  • Node interface is the base interface for all other interfaces in the DOM API
  • Inherits from EventTarget interface for event handling support
  • Defines basic properties and methods for all nodes in DOM tree
  • Document, Element, and CharacterData interfaces extend Node interface
  • Document interface further extended by HTMLDocument interface for HTML documents
  • Element interface further extended by HTMLElement interface for HTML elements
  • CharacterData interface further extended by Text and Comment interfaces for text and comment nodes.
Here we can see that the Node interface is the base interface for all other interfaces in the DOM API. It inherits from the EventTarget interface, which defines the properties and methods that all DOM nodes must implement in order to support event handling. The Node interface defines the basic properties and methods that all nodes in the DOM tree must implement. The Node interface is extended by the Document, Element, and CharacterData interfaces, which define the properties and methods that all document nodes, element nodes, and character data nodes must implement. The Document interface is extended by the HTMLDocument interface, which defines the properties and methods that all HTML documents must implement. The Element interface is extended by the HTMLElement interface, which defines the properties and methods that all HTML elements must implement. The CharacterData interface is extended by the Text and Comment interfaces, which define the properties and methods that all text nodes and comment nodes must implement.

What is an interface?

  • Interfaces are a core concept in programming and software engineering.
  • JavaScript interfaces define the behavior of objects in the DOM API.
  • Interfaces enable polymorphism and code reuse.
  • The Node, Event, and Document interfaces, among others, specify the methods and properties objects must implement in the DOM API.
  • Custom objects can be created with the same interfaces, promoting modularity.
  • Although not a built-in language feature, interfaces are heavily used in the DOM API and other libraries.
  • In this course, we will focus on the DOM API and how to use its interfaces to access and modify the DOM

Interfaces are a core concept in programming and software engineering. They define what a class or object can do without specifying how it does it. They provide a clear separation of concerns between the interface and the implementation, making it easier to maintain and update the codebase.

In the context of JavaScript and the DOM, interfaces define a specification for the behavior of objects in the DOM API.

The DOM API is built upon a set of interfaces, including the Node Interface, the Event Interface, and the Document Interface, among others.

These interfaces define the methods and properties that objects in the DOM API must implement.

By using these interfaces, different objects in the DOM API can be used interchangeably, enabling polymorphism.

Interfaces also allow developers to create their own custom objects that implement the same methods and properties as the objects in the DOM API, enabling greater modularity and code reuse.

In JavaScript, interfaces are not a built-in language feature like they are in Java or C#, but the concept of interfaces is still used extensively in the DOM API and other libraries and frameworks.

The DOM Node Interface

  • The DOM Node Interface is an abstract base class that many other DOM API objects are based on.
  • This allows for object types to be used similarly and interchangeably.

Node subclasses:

  • The Node Interface is an abstract class; there is no plain Node object.
  • All objects that implement Node functionality are based on one of its subclasses, the most notable being Document, Element, and DocumentFragment.
  • Every kind of DOM node is represented by an interface based on Node, including Attribute (Attr), CharacterData(Text, Comment), and DocumentType.

  • Node inherits properties from its parent, EventTarget.
  • These properties provide information about the node's relationship with its parent and children.

The DOM Node Interface Methods

  • Node Interface has several methods, including Node.appendChild(), Node.cloneNode(), Node.compareDocumentPosition(), Node.contains(), Node.getRootNode(), Node.hasChildNodes(), Node.insertBefore(), Node.isDefaultNamespace(), Node.isEqualNode(), Node.isSameNode(), Node.lookupPrefix(), Node.lookupNamespaceURI(), Node.normalize(), Node.removeChild(), and Node.replaceChild().
  • Node inherits methods from its parent, EventTarget.
  • These methods provide ways to manipulate and interact with the node and its children.

The DOM Node Interface methods examples 

const h1 = document.getElementById('h1');

const p = document.createElement('p');
p.textContent = 'A new paragraph.';

h1.appendChild(p);
console.log(h1.childNodes); // logs the updated array-like list of child nodes of the h1 element

const h1Clone = h1.cloneNode(true);
console.log(h1Clone); // logs a clone of the h1 element, including all of its descendants

const docPos = h1.compareDocumentPosition(p);
console.log(docPos); // logs the document position of the p element relative to the h1 element

const containsP = h1.contains(p);
console.log(containsP); // logs true because the h1 element contains the p element

const root = h1.getRootNode();
console.log(root); // logs the root node of the h1 element, which is the HTML document object

const hasChildNodes = h1.hasChildNodes();
console.log(hasChildNodes); // logs true because the h1 element has child nodes

const textNode = document.createTextNode('A new text node.');
h1.insertBefore(textNode, p);
console.log(h1.childNodes); // logs the updated array-like list of child nodes of the h1 element

const defaultNS = h1.isDefaultNamespace('http://www.w3.org/1999/xhtml');
console.log(defaultNS); // logs true because the h1 element is in the XHTML namespace

const h1Equal = h1.isEqualNode(h1Clone);
console.log(h1Equal); // logs false because the h1 element and its clone are not the same node

console.log(h1.childNodes); // logs the updated array-like list of child nodes of the h1 element after removing the text node

h1.replaceChild(p, h1Clone);
console.log(h1.childNodes); // logs the updated array-like list of child nodes of the h1 element after replacing the h1Clone element with the p element

The Document Node Interface

  • The Document interface represents the entire HTML or XML document, and is the root of the document tree.
  • It provides methods and properties to access and manipulate the contents of a document.
  • The Document inherits properties and methods from its parent interface, Node, and by extension, the EventTarget interface.
  • The Document interface also has a number of specific properties and methods, such as documentElement, createElement(), createTextNode(), and querySelector().
  • Other interfaces inherit from Document, such as HTMLDocument and XMLDocument, which provide additional properties and methods specific to HTML and XML documents, respectively.

The Document interface represents the entire HTML or XML document and serves as the root of the document tree. It provides methods and properties to access and manipulate the contents of a document.

The Document inherits properties and methods from its parent interface, Node, which provides methods and properties to work with any type of node. By extension, it also inherits from the EventTarget interface, which allows event listeners to be attached to the document.

The Document interface has a number of specific properties and methods that are used to interact with the document. For example, documentElement represents the root element of the document, and createElement(), createTextNode(), and querySelector() are used to create new elements, text nodes, and to query for elements in the document.

Other interfaces inherit from Document to provide additional functionality specific to certain types of documents. For example, HTMLDocument and XMLDocument provide additional properties and methods specific to HTML and XML documents, respectively.

The Document Node Interface Properties

The Document interface inherits properties from its parent interface, Node, and by extension that interface's parent, EventTarget.

  • URL: returns the document location as a string.
  • anchors: returns a collection of all anchor elements in the document with a name attribute.
  • body: returns the body element of the document.
  • cookie: gets or sets the cookie associated with the document.
  • doctype: returns the DocumentType node of the document.
  • documentElement: returns the root element of the document (usually the HTML element).
  • forms: returns a collection of all form elements in the document.
  • head: returns the head element of the document.
  • images: returns a collection of all img elements in the document.
  • implementation: returns the DOMImplementation object that handles this document.
  • links: returns a collection of all a and area elements in the document with an href attribute.
  • readyState: returns the current state of the document (loading, interactive, complete).
  • referrer: returns the URL of the document that loaded this document.
  • title: gets or sets the title of the document.
  • write(content): writes HTML expressions or JavaScript code to a document.
  • writeln(content): writes the same as write(), but adds a newline character after each statement.

The Document Node Interface Methods

The Document interface inherits methods from its parent interface, Node, and by extension that interface's parent, EventTarget.

Below are the most used methods that are specific to the Document interface.

  • createAttribute(name): creates a new attribute node with the specified name.
  • createComment(data): creates a new comment node with the specified data.
  • createDocumentFragment(): creates a new empty DocumentFragment node.
  • createElement(tagName): creates a new element node with the specified tag name.
  • createEvent(eventInterface): creates a new event of the specified type.
  • createNodeIterator(root, whatToShow, filter, entityReferenceExpansion): creates a new NodeIterator object.
  • createTextNode(data): creates a new text node with the specified data.
  • getElementById(id): returns the element with the specified ID.
  • getElementsByClassName(classNames): returns a collection of elements with the specified class name.
  • getElementsByName(name): returns a collection of elements with the specified name.
  • getElementsByTagName(tagName): returns a collection of elements with the specified tag name.

The Element Node Interface

  • Element is a base class for all element objects in a Document
  • Element also inherits properties and methods from its parent interface, Node, and by extension that interface's parent, EventTarget.
  • More specific classes inherit from Element
  • HTMLElement interface is the base interface for HTML elements
  • SVGElement interface is the basis for all SVG elements
  • Most functionality is specified further down the class hierarchy

The Element class is the most general base class from which all element objects in a Document inherit.

It inherits from the previously discussed Node interface, which provides methods and properties to work with any type of node.

It is the foundation for all elements in a Document.

This class only has methods and properties common to all kinds of elements.

More specific classes inherit from Element, which allows for more specialized functionality and behavior for different types of elements.

Examples of base interfaces include HTMLElement for HTML elements and SVGElement for SVG elements.

Most functionality is specified further down the class hierarchy. This is because the Element class is intended to be a base class for all element objects in a Document, and not a class that is used directly.

The Element Node Interface Properties

Element inherits properties from its parent interface, Node, and by extension that interface's parent, EventTarget. Here are the most used properties that are specific to the Element interface:

  • classList: returns a live DOMTokenList collection of the class attributes of the element.
  • clientHeight: returns the inner height of the element, including padding but not border or margin.
  • clientLeft: returns the width of the element's left border.
  • clientTop: returns the width of the element's top border.
  • clientWidth: returns the inner width of the element, including padding but not border or margin.
  • id: sets or returns the ID of the element.
  • innerHTML: sets or returns the HTML content of the element.
  • outerHTML: sets or returns the HTML content of the element, including the element itself.
  • scrollHeight: returns the height of the scrollable content of the element.
  • scrollLeft: sets or returns the number of pixels that the element's content is scrolled to the left.
  • scrollTop: sets or returns the number of pixels that the element's content is scrolled from the top.
  • scrollWidth: returns the width of the scrollable content of the element.
  • style: returns the inline style of the element as a CSSStyleDeclaration object.
  • tagName: returns the tag name of the element in uppercase letters.

The CharacterData Node Interface

  • The CharacterData interface represents a node containing character data.
  • CharacterData nodes can be accessed and manipulated using the same methods as other types of nodes in the DOM.
  • CharacterData nodes are typically found as children of Element or DocumentType nodes.
  • The Text and Comment interfaces inherit from the CharacterData interface.

Learn more on MDN

The CharacterData interface represents a node containing character data.

CharacterData nodes can be accessed and manipulated using the same methods as other types of nodes in the DOM, such as appendChild(), removeChild(), and replaceChild().

CharacterData nodes are typically found as children of Element or DocumentType nodes. They can also occur as children of other CharacterData nodes, such as in the case of Text and Comment nodes.

The Text and Comment interfaces inherit from the CharacterData interface, which means that they inherit its methods and properties in addition to their own.

JavaScript can be used to access and modify the content of an element:

  • innerHTML : Gets or sets the HTML content of an element (use with caution!) 
     document.getElementById("demo").innerHTML = "Hello JavaScript!";
  • innerText : Gets or sets the visible text content of an element 
     document.getElementById("demo").innerText = "Hello JavaScript!";
  • textContent : Gets or sets the text content of an element, including hidden text 
     document.getElementById("demo").textContent = "Hello JavaScript!";

Accessing Element Styles

JavaScript can be used to manipulate the styles of an element:

  • style : Accesses and modifies the inline styles of an element 
     document.getElementById("demo").style.color = "red";
  • classList : Accesses and modifies the class attribute of an element 
     document.getElementById("demo").classList.add("red") // adds a class
                document.getElementById("demo").classList.remove("red") // removes a class
                document.getElementById("demo").classList.toggle("red") // toggles a class
  • getComputedStyle : Retrieves the computed styles of an element 
     const styles = window.getComputedStyle(document.getElementById("demo")); // retrieves the computed styles
                const color = styles.getPropertyValue("color"); // retrieves the value of a specific style

Accessing Element Dimensions

  • getBoundingClientRect : Retrieves the size and position of an element 
     const rect = document.getElementById("demo").getBoundingClientRect();
  • clientWidth and clientHeight : Retrieves the width/height of an element, including padding but not border 
     const width = document.getElementById("demo").clientWidth;
  • offsetWidth and offsetHeight : Retrieves the width/height of an element, including padding and border 
     const width = document.getElementById("demo").offsetWidth;
  • scrollWidth and scrollHeight : Retrieves the width/height of an element, including padding, border, and scrollbar 
     const width = document.getElementById("demo").scrollWidth;

Introduction to Events

Event are from the EventTarget interface, and are used to handle notifications about things that happen in the DOM.

Events are actions or occurrences that happen in the browser:

  • Clicks
  • Scrolling
  • Mouse movements
  • Keypresses
  • Form submissions
  • And many more...

JavaScript can detect and respond to these events.

Learn more on MDN

The EventTarget interface is implemented by objects that can receive events and may have listeners for them. In other words, any target of events implements the three methods associated with this interface.

Element, and its children, as well as Document and Window, are the most common event targets, but other objects can be event targets, too. For example XMLHttpRequest, AudioNode, and AudioContext are also event targets.

Many event targets (including elements, documents, and windows) also support setting event handlers via onevent properties and attributes.

Event Handlers and Listeners

Event listeners are functions that are called when an event occurs:

  • addEventListener : Attaches an event listener to an element 
    
                  function doSomething() {
                    console.log("Something happened");
                  }
                  element.addEventListener("click", doSomething, false); // false = bubbling, true = capturing, third parameter is optional because it is false by default
  • removeEventListener: Removes an event listener from an element. Requires a reference to the same function 
    
                  element.removeEventListener("click", doSomething, false); // false = bubbling, true = capturing,

You can have multiple listeners for the same event on the same element.

Learn more on MDN

Event Types and Objects

When an event is triggered, an event object is created:

  • Contains information about the event
  • Can be used to control the event's behavior

Some useful properties:

  • type : The type of event
  • target: The element that triggered the event
  • currentTarget: The element that the event listener is attached to
  • preventDefault: Prevents the default action of the event
  • stopPropagation: Stops the event from up or down the DOM (Advanced stuff)

Advanced Event Handling

Passing Parameters to Event Handlers

  • Inline Function: You can create an inline function and pass it to the event listener as a callback. Inside the function, you can pass the necessary parameters. ( Not recommended, because it is difficult to reuse the function ) 

     <button onclick="myFunction('param1', 'param2')">Click me</button>

                <script>
                  function myFunction(param1, param2) {
                    console.log(param1 + ' ' + param2);
                  }
                </script>

  • Use an anonymous function to pass parameters to event handlers:

     element.addEventListener('click', function(event) {
                myFunction(event, param1, param2);
              });

  • Event Object: When an event is triggered, an Event object is passed to the event handler function. 

     function myFunction(event) {
                console.log(event.target.id);
              }
              
              document.getElementById('myButton').addEventListener('click', myFunction);

  • Function Reference: You can also pass a reference to a function and use the bind() method to pass parameters. 

     function myFunction(param1, param2) {
                console.log(param1 + ' ' + param2);
              }
              
              document.getElementById('myButton').addEventListener('click', myFunction.bind(null, 'param1', 'param2'))

Learn more on MDN

Event Propagation and Bubbling

Events propagate through the DOM in two phases:

  • Capturing phase: The event travels down from the root to the target element
  • Bubbling phase: The event bubbles up from the target element to the root

Use event.stopPropagation() to prevent further propagation of the current event in bubbling and capturing phases:


        element.addEventListener('click', function(event) {
          event.stopPropagation();
        });

Learn more on MDN

Event propagation is a very important concept in JavaScript. It is used in many libraries and frameworks like React, Angular, Vue, etc.

Working with Different Handler Functions and Scopes

Event handlers can be:

  • Inline event handlers (not recommended)
  • Named functions
  • Anonymous functions

Event handlers can have different scopes:

  • Global scope
  • Function scope
  • Lexical scope

Use bind() to change the value of this in event handlers:


        element.addEventListener('click', myFunction.bind(myObject));
